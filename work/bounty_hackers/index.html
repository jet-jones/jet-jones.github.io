I really wouldn't recommended using unity, hdrp in particular if you're trying do a visually interesting 3d game. I think it's probably fine if you just want to have a single shader effect cel shading, outlines etc that drives your artstyle or it's mostly unlit but if you're trying to experiment in engine and find new interesting visuals, unity just does not have the foundations in place to support experimentation driven development. Shaders often take several minutes to couple, and it's almost impossible to code shaders directly so you have to resort to strange hacks with custom functions to insert things like instancing into the compiled code, documentation is severely limited for the newer pipelines. even the community that's one of the main reasons people have chosen unity historically Maybe the built in pipeline solves this, maybe theres some other shader editor that fixes these issues but the fact that to be able to iterate quickly on shaders, post processing etc. even just basic object placement has severe issues so I ended up doing most of my work in modelling programs and then exporting as a single mesh which limited my ability to see what the final scene was going to look like.
